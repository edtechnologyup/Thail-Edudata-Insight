# แผนการทำงานของระบบ Data Catalog การศึกษาไทย

---

## ภาพรวม

ระบบ Data Catalog สำหรับรวบรวมข้อมูลการศึกษาไทยจากหลายหน่วยงาน เช่น สพฐ., สอศ., สกศ., มหาวิทยาลัย
ทำด้วย AI 100% โดยใช้ Cursor + Claude ตาม Spec ที่กำหนด

---

## ขั้นตอนที่ 1 — UX/UI Mockup

> เห็นภาพก่อน Build จริง

- ออกแบบ Layout ทุกหน้าตาม Spec #35 #36 #38
- ใช้ Design System ตาม Spec #41 (สี, Typography, Spacing)
- ครอบคลุมทุก Role: Public / Agency / Admin
- ส่งให้ Stitch ออกแบบต่อ

**หน้าที่ต้องออกแบบ**

- Public: หน้าหลัก, ค้นหา, รายละเอียด Dataset, สถิติ, Privacy Policy
- Auth: Login, Register
- Agency: Dashboard, อัปโหลด Dataset, รายการ Dataset, Version History, Bookmark, Subscription, จัดการหมวดหมู่ระดับ 2 ของตัวเอง
- Admin: Dashboard, จัดการ User, Approve Dataset, จัดการหมวดหมู่ระดับ 1/แท็ก (แสดงแบบ Tree 2 ระดับ), ประกาศ, Audit Log

## ขั้นตอนที่ 3 — Database

> ต้องพร้อมก่อน Backend เขียน API ได้

- สร้าง Migration ทุกตาราง ตาม Spec #11 #12
  - users, datasets, dataset_versions, dataset_files
  - categories (มี parent_id, level, created_by รองรับ 2 ระดับ), tags, dataset_tags
  - bookmarks, subscriptions, saved_searches
  - download_logs, dashboard_layouts, announcements, audit_logs, pdpa_consents
- ตาราง categories ต้องมีครบดังนี้
  - `parent_id` → null = ระดับ 1, มีค่า = ระดับ 2
  - `level` → 1 หรือ 2
  - `created_by` → FK → users.id (รู้ว่า Admin หรือ Agency เป็นคนสร้าง)
- กำหนด Enum ทุกตัว ตาม Spec #14
- สร้าง Index ตาม Spec #17 รวมถึง `idx_categories_parent_id` และ `idx_categories_level`
- ตั้งค่า Soft Delete ตาม Spec #15
- ตรวจสอบ ER Diagram ตาม Spec #19
- ตั้งกฎ Migration ตาม Spec #18

## ขั้นตอนที่ 6 — M2 · Dataset Module

> หัวใจหลักของระบบ

- Upload Dataset Flow ตาม Spec #29
- ช่องเลือกหมวดหมู่ในฟอร์มต้องเลือกได้ถึงระดับ 2 (Cascading Select)
- File Validation (CSV/Excel/JSON, ไม่เกิน 100MB) ตาม Spec #45
- PII Masking อัตโนมัติ ตาม Spec #46
- Data Quality Score คำนวณด้วย Pandas
- Approval Workflow (Submit → Approve / Reject) ตาม Spec #30
- Version History และ Restore ตาม Spec #5 (M2)
- Bulk Upload ด้วย Excel Template ตาม Spec #29
- Transaction / Rollback Rules ตาม Spec #34
- Agency จัดการหมวดระดับ 2 ของตัวเอง ตาม Spec #20 #5 (หมวดหมู่)
  - เพิ่มหมวดระดับ 2 ใต้ระดับ 1 ที่ Admin สร้าง
  - แก้ไขได้เฉพาะหมวดระดับ 2 ที่ตัวเองสร้าง
  - ลบได้เฉพาะเมื่อไม่มี Dataset อยู่ใต้นั้น
- ทดสอบ Business Rules ตาม Spec #5 (M2) และ Spec #5 (หมวดหมู่)
